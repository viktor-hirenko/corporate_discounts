## 🚀 Швидкий старт

### Що таке Admin Panel?

Admin Panel — це інструмент для управління контентом сайту **Corporate Discounts**. Тут ви можете:

- ✅ Додавати, редагувати та видаляти партнерів
- ✅ Керувати категоріями та локаціями (які з'являються у фільтрах на сайті)
- ✅ Оновлювати FAQ
- ✅ Редагувати тексти на сайті
- ✅ Керувати доступом користувачів (whitelist)
- ✅ Зберігати зміни на сервер

### Хто має доступ?

Доступ до Admin Panel мають **тільки** користувачі, додані до списку дозволених (whitelist). Для входу використовується **Google акаунт** вашої корпоративної пошти.

### Як отримати доступ?

Зверніться до адміністратора системи, щоб вас додали до списку користувачів у вкладці "Користувачі".

## 🤝 Партнери — Управління партнерами

```

### 📝 Додати нового партнера

#### Крок 1: Відкрийте форму

Натисніть кнопку **"+ Додати партнера"** у правому верхньому куті.

#### Крок 2: Заповніть обов'язкові поля

| Поле              | Опис                               | Приклад     |
| ----------------- | ---------------------------------- | ----------- |
| **Назва (UA)** ⚠️ | Назва партнера українською         | "Roślynka"  |
| **Назва (EN)** ⚠️ | Назва партнера англійською         | "Roslynka"  |
| **Slug**          | Генерується автоматично з EN назви | "roslynka"  |
| **Промокод** ⚠️   | Код знижки для користувачів        | "UPSTARS10" |
| **Категорія** ⚠️  | Вибір зі списку категорій          | "Здоров'я"  |
| **Локація** ⚠️    | Вибір зі списку локацій            | "UA/Київ"   |

> ⚠️ = обов'язкове поле

#### Крок 3: Заповніть деталі знижки

| Поле                    | Опис                     | Приклад                     |
| ----------------------- | ------------------------ | --------------------------- |

| **Розмір знижки (EN)**  | Текст знижки англійською | "-10%" або "500 UAH off"    |
| **Опис знижки (UA/EN)** | Додаткові умови          | "На перше замовлення"       |

| **Повний опис (UA/EN)**   | Детальний опис | На сторінці партнера          |

#### Крок 5: Заповніть контакти

| Поле               | Формат | Приклад                     |
| ------------------ | ------ | --------------------------- |
| **Вебсайт**        | URL    | "https://roslynka.com"      |
| **Email**          | Email  | "info@roslynka.com"         |
| **Телефон**        | Текст  | "+380 44 123 4567"          |
| **Адреса (UA/EN)** | Текст  | "м. Київ, вул. Хрещатик, 1" |

#### Крок 6: Додайте соціальні мережі

| Мережа        | Формат                                      |
| ------------- | ------------------------------------------- |
| **Facebook**  | Повний URL: `https://facebook.com/partner`  |
| **Instagram** | Повний URL: `https://instagram.com/partner` |

#### Крок 7: Додайте умови використання

Натисніть **"+ Додати"** для кожної умови:

```
✓ Знижка діє на всі товари
✓ Необхідно назвати промокод UPSTARS10
✓ Знижка не сумується з іншими акціями
```

#### Крок 8: Збережіть

Натисніть **"Створити партнера"**.

> 💡 **Автозбереження:** Після створення партнера зміни автоматично зберігаються на сервер.

### ⚠️ Важливо: Зв'язок з фільтрами на сайті

Категорії, які ви створюєте тут, **автоматично з'являються** у:

- Фільтрі "Категорія" на сторінці каталогу

```
┌─────────────────────────────────────────────────────────────────┐

Деякі категорії позначені як **"Системна"** і не можуть бути видалені:

| ID       | Назва         | Призначення                           |

## 📍 Локації — Управління локаціями

### ⚠️ Важливо: Зв'язок з фільтрами на сайті

Локації працюють аналогічно категоріям. Вони з'являються у:

- Фільтрі "Локація" на сторінці каталогу
- Випадаючому списку "Локація" при створенні/редагуванні партнера

### Формат локацій

```
Код_країни/Місто
```

**Приклади:**

- `UA/Київ` — Україна, Київ
- `UA/Онлайн` — Україна, онлайн-сервіс
- `PL/Варшава` — Польща, Варшава
- `Online` — Глобальний онлайн

### Логіка фільтрації на сайті

| Фільтр      | Що показує                                |
| ----------- | ----------------------------------------- |
| **Україна** | Партнери з `UA/*`                         |
| **Європа**  | Партнери з `PL/*`, `LT/*`, `CZ/*` та інші |
| **Онлайн**  | Партнери з `Online` або `*/Онлайн`        |

### Системні локації

| ID       | Назва       | Призначення            |
| -------- | ----------- | ---------------------- |
| `all`    | Всі локації | Показує всіх партнерів |
| `online` | Онлайн      | Онлайн-партнери        |

---

### ➕ Додати локацію

1. Натисніть **"+ Додати локацію"**
2. Заповніть:

| Поле              | Опис                     | Приклад    |
| ----------------- | ------------------------ | ---------- |
| **ID (slug)**     | Унікальний ідентифікатор | "ua-lviv"  |
| **Назва (UA)** ⚠️ | Назва українською        | "UA/Львів" |
| **Назва (EN)**    | Назва англійською        | "UA/Lviv"  |

3. Натисніть **"Створити"**

## 👥 Користувачі — Управління доступом

### Що таке whitelist?

Це список email-адрес, яким **дозволено входити** в Admin Panel через Google авторизацію. Якщо email немає в списку — користувач побачить помилку "Доступ заборонено".

### Ролі користувачів

| Роль              | Доступ                                                   |
| ----------------- | -------------------------------------------------------- |
| **Адміністратор** | Повний доступ, включаючи управління користувачами        |
| **Редактор**      | Редагування контенту, без доступу до списку користувачів |

---

### ➕ Додати користувача

1. Натисніть **"+ Додати користувача"**
2. Заповніть:

| Поле         | Опис             | Приклад               |
| ------------ | ---------------- | --------------------- |
| **Email** ⚠️ | Google email     | `manager@upstars.com` |
| **Ім'я**     | Ім'я користувача | "Іван Петренко"       |
| **Роль**     | admin або editor | "Редактор"            |

3. Натисніть **"Додати"**

> **⚠️ Важливо:** Email має бути Google акаунтом!

---

### 💾 Зберегти зміни на сервер

Після додавання/редагування/видалення користувачів:

1. Натисніть **"Зберегти зміни на сервер"** внизу сторінки
2. Дочекайтеся підтвердження

### Як працює збереження

```
┌──────────────────────────────────────────────────────────────┐
│ 1. Ви редагуєте дані в Admin Panel                            │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 2. Натискаєте "Зберегти зміни на сервер"                     │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 3. Дані надсилаються на Cloudflare Worker                    │
│    POST /api/save-config                                      │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 4. Worker зберігає app-config.json в R2                       │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ 5. Зміни доступні на сайті після оновлення сторінки          │
└──────────────────────────────────────────────────────────────┘
```

### Автозбереження

Деякі дії автоматично зберігають зміни:

- ✅ Створення партнера
- ✅ Редагування партнера
- ✅ Видалення партнера
- ✅ Дублювання партнера

### Ручне збереження

Для інших розділів потрібно натиснути **"Зберегти зміни на сервер"**:

- Dashboard (панель експорту)
- Користувачі (кнопка внизу)

### 📥 Експорт в файл

Для бекапу даних:

1. Перейдіть на **Dashboard**
2. Натисніть **"Експорт в файл"**
3. JSON файл буде завантажено на комп'ютер

## ❓ FAQ для менеджерів

### Загальні питання

**Q: Як часто потрібно зберігати зміни?**  
A: Партнери зберігаються автоматично. Для користувачів — натисніть "Зберегти" після змін.

**Q: Чи можуть двоє працювати одночасно?**  
A: Так, але будьте обережні! Останній, хто збереже, перезапише зміни першого.

**Q: Що робити, якщо випадково видалив партнера?**  
A: Якщо зміни ще не застосовані — оновіть сторінку. Якщо вже збережено — зверніться до розробника.

---

### Партнери

**Q: Чому новий партнер не з'являється на сайті?**  
A:

1. Перевірте чи заповнені всі обов'язкові поля
2. Перевірте чи збережені зміни (автоматично після створення)
3. Оновіть сторінку сайту (Ctrl+Shift+R)

**Q: Як додати партнера без промокоду?**  
A: Промокод обов'язковий. Вкажіть "UPSTARS" або "NODISCOUNT".

**Q: Чи можна змінити slug партнера?**  
A: Ні. Створіть нового партнера з правильним slug і видаліть старого.

---

### Категорії та локації

**Q: Що станеться з партнерами, якщо видалити категорію?**  
A: Партнери залишаться, але матимуть неіснуючу категорію. Це може спричинити проблеми з фільтрацією.

**Q: Як дізнатися, які партнери використовують категорію?**  
A: Перейдіть в "Партнери" і виберіть фільтр по цій категорії.

---

### Користувачі

**Q: Що робити, якщо колега не може увійти?**  
A:

1. Перевірте чи його email є в списку користувачів
2. Переконайтеся, що email написаний правильно
3. Перевірте чи збережені зміни на сервер

**Q: Можна додати Gmail акаунт?**  
A: Так, якщо в Google Cloud Console дозволені зовнішні користувачі.

## 📄 Структура конфігурації (app-config.json)

Цей розділ описує всі поля конфігурації сайту в зрозумілому форматі.

### Загальні налаштування

| Поле            | Опис                  | Приклад        |
| --------------- | --------------------- | -------------- |
| `locales`       | Доступні мови сайту   | `["ua", "en"]` |
| `defaultLocale` | Мова за замовчуванням | `"ua"`         |

### Користувачі (`allowedUsers`)

Список email-адрес з доступом до адмін-панелі.

| Поле      | Опис                       | Приклад                   |
| --------- | -------------------------- | ------------------------- |
| `id`      | Унікальний ідентифікатор   | `"user-001"`              |
| `email`   | Google email користувача   | `"manager@upstars.com"`   |
| `name`    | Ім'я користувача           | `"Олександр Петренко"`    |
| `role`    | Роль: `admin` або `editor` | `"admin"`                 |
| `addedAt` | Дата додавання             | `"2025-11-20"`            |
| `addedBy` | Хто додав                  | `"v.hirenko@upstars.com"` |

### Партнери (`partners`)

Кожен партнер — це об'єкт з ключем = slug партнера.

| Поле                                | Опис                       | Приклад                            |
| ----------------------------------- | -------------------------- | ---------------------------------- |
| `id`                                | Унікальний ID партнера     | `"roslynka"`                       |
| `slug`                              | URL-ідентифікатор          | `"roslynka"`                       |
| `image`                             | Шлях до логотипу           | `"/images/partners/roslynka.webp"` |
| `promoCode`                         | Промокод для знижки        | `"UPSTARS10"`                      |
| `name.ua` / `name.en`               | Назва партнера             | `"Рослинка"` / `"Roslynka"`        |
| `category.ua` / `category.en`       | Категорія                  | `"Здоров'я"` / `"Health"`          |
| `location.ua` / `location.en`       | Локація                    | `"UA/Київ"` / `"UA/Kyiv"`          |
| `summary.ua` / `summary.en`         | Короткий опис (картка)     | `"Магазин кімнатних рослин"`       |
| `description.ua` / `description.en` | Повний опис (сторінка)     | `"Детальний опис..."`              |
| `discount.label.ua` / `.en`         | Текст знижки               | `"-10%"`                           |
| `discount.description.ua` / `.en`   | Опис знижки                | `"На перше замовлення"`            |
| `contact.website`                   | URL сайту                  | `"https://roslynka.com"`           |
| `contact.email`                     | Email партнера             | `"info@roslynka.com"`              |
| `contact.phone`                     | Телефон                    | `"+380 44 123 4567"`               |
| `address.ua` / `address.en`         | Адреса                     | `"м. Київ, вул. Хрещатик, 1"`      |
| `socials`                           | Масив соц. мереж           | `[{type: "facebook", url: "..."}]` |
| `terms.ua` / `terms.en`             | Умови використання (масив) | `["Знижка на всі товари", "..."]`  |
| `tags.ua` / `tags.en`               | Теги (опціонально)         | `["рослини", "подарунки"]`         |

### Категорії (`filters.categories`)

| Поле                     | Опис                         | Приклад                   |
| ------------------------ | ---------------------------- | ------------------------- |
| `id`                     | Унікальний ID                | `"health"`                |
| `name.ua` / `name.en`    | Назва категорії              | `"Здоров'я"` / `"Health"` |
| `description.ua` / `.en` | Опис (опціонально)           | `"Медичні послуги"`       |
| `isSystem`               | Чи системна (не видаляється) | `true` / `false`          |

**Системні категорії:** `all` (всі), `online` (онлайн).

### Локації (`filters.locations`)

| Поле                  | Опис                         | Приклад                   |
| --------------------- | ---------------------------- | ------------------------- |
| `id`                  | Унікальний ID                | `"ua-kyiv"`               |
| `name.ua` / `name.en` | Назва локації                | `"UA/Київ"` / `"UA/Kyiv"` |
| `isSystem`            | Чи системна (не видаляється) | `true` / `false`          |

**Системні локації:** `all` (всі), `online` (онлайн).

### FAQ (`pages.faq.items`)

| Поле                  | Опис                 | Приклад                 |
| --------------------- | -------------------- | ----------------------- |
| `id`                  | Унікальний ID        | `"faq-001"`             |
| `category`            | Категорія питання    | `"general"`             |
| `question.ua` / `.en` | Текст питання        | `"Як отримати знижку?"` |
| `answer.ua` / `.en`   | Текст відповіді      | `"Назвіть промокод..."` |
| `order`               | Порядок відображення | `1`, `2`, `3`...        |

### Зображення (`images`)

| Ключ              | Опис                     | Шлях                                   |
| ----------------- | ------------------------ | -------------------------------------- |
| `logoDark`        | Темний логотип (header)  | `/images/upstars-logo-dark.svg`        |
| `logoLight`       | Світлий логотип (footer) | `/images/upstars-logo-light.svg`       |
| `tagline`         | Тегляйн                  | `/images/corporate-discounts-text.svg` |
| `loginBackground` | Фон сторінки входу       | `/images/login-background.svg`         |
| `botImage`        | Зображення бота (FAQ)    | `/images/bot-img.svg`                  |

### Тексти (`pages`, `auth`, `navigation`, `pagination`)

Всі тексти мають структуру `{ ua: "...", en: "..." }`.

**Приклад:**

```json
"pages": {
  "discounts": {
    "title": {
      "ua": "Корпоративні знижки",
      "en": "Corporate Discounts"
    },
    "noResults": {
      "ua": "Нічого не знайдено",
      "en": "Nothing found"
    }
  }
}
```

### Як це працює разом?

```
┌───────────────────────────────────────────────────────────────┐
│                     app-config.json                           │
├───────────────────────────────────────────────────────────────┤
│  allowedUsers[] ──────────────► Хто може входити в адмінку   │
│  partners{}     ──────────────► Картки партнерів на сайті     │
│  filters.categories[] ────────► Кнопки фільтрів "Категорія"   │
│  filters.locations[]  ────────► Кнопки фільтрів "Локація"     │
│  pages.faq.items[]    ────────► Питання/відповіді на FAQ      │
│  images{}       ──────────────► Логотипи та фони              │
│  pages/auth/navigation ───────► Всі тексти на сайті           │
└───────────────────────────────────────────────────────────────┘
```
